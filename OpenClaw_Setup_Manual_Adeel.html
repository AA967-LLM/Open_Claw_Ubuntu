01. Target System Audit

The following hardware configuration was verified on the master system. New deployment targets should match or exceed these specs for comparable performance.

CPU i9-13980HX (24C/32T)
GPU RTX 4080 (12GB VRAM)
Memory 32GB DDR5
OS Host Windows 11 / WSL2
Note on GPU: Ensure NVIDIA Driver 590+ is installed on the Windows host to support the latest CUDA kernels used by Step3-VL.

02. Windows 11 Host Setup

  • 1 Enable WSL2: Open PowerShell as Admin and run: 
    wsl --install -d Ubuntu-24.04
  • 2 PowerShell Execution Policy: Allow automation scripts: 
    Set-ExecutionPolicy RemoteSigned -Scope CurrentUser -Force
  • 3 Install Chocolatey (Package Manager): 
    Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
  • 4 Essential Host Tools: 
    choco install git nodejs python ollama -y

03. WSL2 (Linux) Agent Setup

Switch to your Ubuntu terminal for these commands.

  • 1 Install OpenClaw Globally: 
    npm install -g openclaw
  • 2 Initialize Workspace: 
    openclaw setup
  • 3 The Local Model Serve Trick: Create a virtual environment for the heavy vision models to avoid dependency conflicts. 
    python3 -m venv step3-venv
source step3-venv/bin/activate
pip install torch transformers fastapi uvicorn pillow

05. WhatsApp & Brain Config

Modify ~/.openclaw/openclaw.json to link your phone and set the primary "Brain" (DeepSeek).

{
  "agents": {
    "defaults": {
      "model": {
        "primary": "ollama/deepseek-v3.1:671b-cloud"
      }
    }
  },
  "channels": {
    "whatsapp": { "enabled": true }
  }
}

To Link WhatsApp: Run openclaw channels login and scan the generated QR code.

06. Critical Fixes (The "Adeel Protocol")

  • Context Window Error: If you see "Something went wrong", it is likely a model window mismatch. DeepSeek via Ollama is the preferred "Brain" for long conversations.
  • Port Conflicts: If the gateway won't start, run openclaw gateway --force to kill stale processes on port 18789.
  • WSL2 IP Changes: If the agent can't see the Windows host, use $(cat /etc/resolv.conf | grep nameserver | awk '{print $2}') to find the host IP.
